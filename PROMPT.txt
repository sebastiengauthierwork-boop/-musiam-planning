Creer les utilisateurs directement depuis l'app sans passer par le dashboard Supabase :

1. CREER UNE API ROUTE Next.js : app/api/create-user/route.ts
- Cette route recoit en POST : email, password, role, employee_id, allowed_teams, allowed_site_id
- Elle utilise le SUPABASE SERVICE ROLE KEY (pas la anon key) pour creer le user dans auth.users avec supabase.auth.admin.createUser()
- Le service role key permet de creer un user sans email de confirmation
- Puis elle insere dans la table users avec le bon role, employee_id, etc.
- Retourne success ou erreur

2. VARIABLE D'ENVIRONNEMENT :
- Ajouter SUPABASE_SERVICE_ROLE_KEY dans les variables d'environnement Vercel (pas NEXT_PUBLIC, c'est cote serveur uniquement)
- Dans le code, utiliser process.env.SUPABASE_SERVICE_ROLE_KEY
- Genere les instructions pour recuperer la service role key dans Supabase : Settings > API > service_role key

3. SECURITE :
- La route /api/create-user doit verifier que l'appelant est un admin ou responsable connecte (verifier le token auth)
- Ne JAMAIS exposer le service_role key cote client
- La route est accessible uniquement en POST avec un token valide

4. PAGE /admin/utilisateurs :
- Modifier le formulaire pour appeler /api/create-user au lieu de supabase.auth.signUp()
- Le champ mot de passe est obligatoire
- Plus besoin de passer par le dashboard Supabase
