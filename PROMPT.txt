Possibilite de modifier l'email d'un utilisateur depuis /admin/utilisateurs :

1. Dans le modal de modification d'un utilisateur, rendre le champ email modifiable UNIQUEMENT si l'utilisateur connecte est admin. Si c'est un responsable ou manager, le champ email est affiche mais non modifiable (disabled).

2. Quand l'admin modifie l'email et clique Enregistrer :
   a) Appeler une nouvelle API route POST /api/update-user avec : user_id, new_email
   b) Cette route verifie que l'appelant est admin (pas responsable, pas manager)
   c) Utilise SUPABASE_SERVICE_ROLE_KEY et supabase.auth.admin.updateUserById(user_id, { email: new_email, email_confirm: true })
   d) Met a jour l'email dans la table users
   e) Met a jour l'email dans la table employees si le user a un employee_id

3. Message de succes : "Email modifie. Le salarie doit maintenant se connecter avec [nouvel email]"

git add . && git commit -m "feat: modification email par admin uniquement" && git push origin main
